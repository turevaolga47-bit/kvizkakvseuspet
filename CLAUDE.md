## Что это за проект
Квиз-лендинг для психолога Ольги Турьевой. Название: «Как всё успеть и не разорваться на части».
Цель: лидогенерация — пользователь проходит квиз, получает результат по сегменту выгорания (low / medium / high) и попадает в форму записи на консультацию.

---

## Правила работы с проектом

### Структура
- Всё приложение — один SPA без роутера. Новые экраны добавляются как состояния в `src/App.tsx`, не как отдельные страницы или маршруты.
- Не добавлять новые npm-зависимости без явной необходимости. Всё, что можно сделать на React + Tailwind + встроенных браузерных API — делаем так.

### Дизайн
Палитра проекта фиксирована — не менять без согласования:

| Роль | Цвет |
|------|------|
| Фон страниц | `#FDF7F3` (тёплый кремовый) |
| Основной акцент | `#E9A6B2` (нежно-розовый) |
| Акцент hover | `#E38C9C` |
| Основной текст | `#2F3337` |
| Вторичный текст | `#6B7280` |
| Заглушки | `#9CA3AF` |

- Скругления: `rounded-2xl` / `rounded-3xl` / `rounded-full`
- Шрифт: Inter (уже подключён глобально)
- Анимации только из `index.css` (`animate-fade-in`, `animate-slide-up`) — не добавлять новые библиотеки анимаций

### Адаптивность
- Mobile-first. Все новые экраны и компоненты должны корректно работать на экранах от 375px.
- Breakpoints Tailwind: `sm:` (640px) и `lg:` (1024px).
- Модалки на мобайле — снизу (`items-end`), на десктопе — по центру (`sm:items-center`).
- Кнопки и тапабельные элементы — минимум 44px по высоте.

### Стиль текста
- Простой и дружеский — как разговор, а не инструкция.
- Без канцелярита: не «осуществить», «произвести», «в рамках» — а «сделать», «провести», «в».
- Обращение к пользователю — на «вы» (с маленькой буквы), в женском роде.

### Процесс
- **Перед созданием нового файла** — сначала описать, что планируется сделать и зачем, и дождаться подтверждения.
- Перед правкой существующего компонента — прочитать его целиком.
- Не добавлять комментарии в код без явной просьбы.

## Стек
- React 18 + TypeScript + Vite
- Tailwind CSS (без дополнительных плагинов)
- Деплой: Railway (`npm run build` → `npx serve dist`)
- Интеграции: Google Sheets API, EmailJS, Telegram Bot API
- Секреты в `.env`

## Команды
```bash
npm run dev      # локальный дев-сервер
npm run build    # tsc + vite build → dist/
npm run preview  # предпросмотр dist/
```

## Структура приложения
Флоу экранов в `src/App.tsx`:
```
start → quiz → analyzing → form → result
```
Хэш `#admin` открывает AdminPanel (панель просмотра заявок).

| Файл | Назначение |
|------|-----------|
| `src/App.tsx` | Флоу + состояние квиза |
| `src/data/questions.ts` | Вопросы и значения ответов |
| `src/utils/scoring.ts` | Подсчёт баллов → сегмент |
| `src/utils/api.ts` | Google Sheets / EmailJS / Telegram |
| `src/components/BookingForm.tsx` | Модалка записи на консультацию |
| `src/components/CalendarPicker.tsx` | Выбор даты и времени |
| `src/components/QuizQuestion.tsx` | Отображение вопроса |
| `src/components/AdminPanel.tsx` | Просмотр заявок (`#admin`) |
| `src/components/FormGate.tsx` | Сбор контактов перед результатом |
| `src/components/ResultScreen.tsx` | Экран результата (3 сегмента) |

## Важные детали
- Нет роутера — навигация через `useState<Screen>`
- Сегменты результата: `low` / `medium` / `high` (по сумме баллов из `src/utils/scoring.ts`)
- Типы: `Screen`, `QuizAnswer`, `ResultSegment` — в `src/types/index.ts`
- `BookingForm` принимает полный payload с ответами, баллом и сегментом
