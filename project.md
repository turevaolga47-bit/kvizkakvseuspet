# Проект: Квиз «Как всё успеть и не разорваться на части»

## Что это за проект

Квиз-лендинг для кризисного психолога **Ольги Турьевой** (34 года практики, работает онлайн из Чехии).
Пользователь отвечает на 6 вопросов о своей усталости и ощущении «должна» — и получает персональный разбор от психолога с рекомендациями. Основная цель — лидогенерация: провести женщину от осознания проблемы к записи на бесплатную 30-минутную консультацию.

---

## Для кого

Целевая аудитория — **женщины**, которые чувствуют себя «женщиной-оркестром»: тянут работу, семью, быт и при этом живут в режиме «надо», а не «хочу». Текст квиза и результатов написан в женском роде.

Сегментация по итогам квиза (сумма баллов 0–18):
| Сегмент | Описание | Баллы |
|---------|----------|-------|
| **high** | Много «должна», мало ресурса | высокие |
| **medium** | Усталость заметна, ресурс ещё есть | средние |
| **low** | Лёгкая усталость | низкие |

## Страницы / экраны (уже есть)

Навигация — без роутера, через `useState<Screen>` в `App.tsx`.

| Экран | Компонент | Что происходит |
|-------|-----------|----------------|
| **Старт** | `StartScreen` | Лендинг: фото Ольги + коллаж, заголовок, кнопка «Пройти 6 вопросов», ссылки на Telegram-канал / YouTube / сайт |
| **Квиз** | `QuizQuestion` | 6 вопросов с 4 вариантами ответа, прогресс-бар, анимация смены вопросов, фото к каждому вопросу, кнопка «Назад» |
| **Анализ** | `AnalyzingScreen` | Заставка «Анализируем ваши ответы…» на 2.6 секунды |
| **Форма-шлюз** | `FormGate` | Открывает Google Form в новой вкладке; кнопка «Я заполнила» появляется только после клика — так собираются контакты до показа результата |
| **Результат** | `ResultScreen` | Персональный разбор по сегменту: что сейчас происходит, что уже хорошо, риски, 3 шага на неделю, приглашение на встречу, CTA-кнопки (Telegram, канал, сайт) |
| **Форма записи** | `BookingForm` | Модальное окно: имя, телефон/Telegram, email, выбор даты и времени через `CalendarPicker`, тема встречи |
| **Админ-панель** | `AdminPanel` | Скрытый маршрут `#admin`: вход по паролю, вкладки — Статистика / Заявки / Рассылка |

### Дополнительные компоненты
- `CalendarPicker` — визуальный выбор даты + слота времени (Пн–Чт 10–19, Пт 15–17, выходные закрыты)
- `ProgressBar` — прогресс по вопросам
- `AnswerCard` — карточка варианта ответа
- `PrivacyFooter` — футер с политикой конфиденциальности (GDPR, Чехия)

## Стиль дизайна

**Тёплый минимализм** — мягкая женская эстетика без перегруза.

### Палитра
| Роль | Цвет | Hex |
|------|------|-----|
| Фон страниц | Тёплый кремовый | `#FDF7F3` |
| Основной акцент | Нежно-розовый | `#E9A6B2` |
| Акцент тёмный (hover) | Чуть насыщеннее | `#E38C9C` |
| Основной текст | Тёмно-серый | `#2F3337` |
| Вторичный текст | Серый | `#6B7280` |
| Заглушки / метки | Светло-серый | `#9CA3AF` |
| Фон форм / карточек | Белый | `#FFFFFF` |
| Лёгкий розовый фон | Акцентный блок | `#FDF2F4` |

### Типографика
- Шрифт: **Inter** (system-ui fallback)
- Заголовки: `clamp()` — адаптивный размер от 22px до 40px
- Текст: 15–16px, line-height 1.7–1.75

### UI-элементы
- Скругления: `rounded-2xl` (16px), `rounded-3xl` (24px), `rounded-full` для кнопок и бейджей
- Кнопки: розовый фон + белый текст + `box-shadow` с розовым свечением; hover меняет оттенок inline-стилем
- Анимации: `animate-fade-in`, `animate-slide-up` (определены в `index.css`)
- Фото: Pexels / Unsplash, без alt-текста (декоративные)
- Никаких иконок-библиотек — SVG инлайном

### Адаптивность
- Mobile-first, Tailwind breakpoints `sm:` и `lg:`
- Старт-экран: одна колонка на мобайле, две (`lg:flex-row`) на десктопе
- Модалки: снизу на мобайле (`items-end`), по центру на десктопе (`sm:items-center`)
